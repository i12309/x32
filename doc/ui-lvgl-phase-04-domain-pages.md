# Этап 04: перенести рабочие и сервисные страницы

## Цель

Перенести страницы, которые управляют заданиями, профилями, процессом, сервисными механизмами и статистикой.

## Группы страниц

1. Задания:
   - `TaskRun`
   - `TaskProcess`
   - `Task`
   - `List`
2. Профили:
   - `Profile`
   - `List` в режиме профилей или отдельный adapter состояния списка.
3. Сервис:
   - `Service`
   - `Service2`
   - `Table`
   - `Paper`
   - `Guillotine`
   - `Throws`
   - `Bigel`
   - `Calibration`
   - `Slice`
4. Сеть, обновления, статистика:
   - `Wifi`
   - `Stats`
   - `Update`
   - `Page`

## Работы

1. Для каждой страницы создать класс в `src/Screen/Page/<Name>.h|cpp`.
2. Перенести старые callbacks из `src/UI/**/p*.cpp` в методы классов.
3. Заменить прямые `NexText::setText`, `getText`, `Set_font_color_pco`, `Set_background_color_bco` на wrapper API.
4. Перенести page mode/back mode из `Catalog::PageMode` в новый навигационный контекст.
5. Для повторяемого `List` сделать отдельный `ListModel`/`ListPageMode`, чтобы одна generated страница могла показывать задания, профили или статистические списки.
6. Для сервисных экранов вынести повторяемые индикаторы датчиков/кнопок в `src/Screen/Panel`, если они требуют общего графического состояния.
7. Удалять старую страницу только после того, как все входящие ссылки на нее заменены.

## Проверки

- Сценарий запуска задания:
  - выбрать задание;
  - выбрать профиль;
  - изменить cycles;
  - стартовать процесс;
  - перейти на `TaskProcess`.
- Сервисные кнопки вызывают те же state/registry действия, что и старый UI.
- `Stats`, `Wifi`, `Update` не используют Nextion UART.
- `pio run -e JC8048W550C` проходит после каждой группы страниц.

## Риски

- Старый UI использует `checkButton("START", ...)` для физических кнопок. Это поведение нужно сохранить отдельно от touch callbacks.
- Некоторые страницы читают состояние из цвета/видимости Nextion-объектов. В LVGL это нужно заменить на явное состояние в page class/model.
- `List` сейчас выглядит как универсальная страница, но логика в старом коде может быть раскидана по `pTaskList`, `pProfileList`, `pShowStat`.
